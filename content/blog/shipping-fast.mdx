My hard drive has a graveyard.

Folders with names like `project-x-v2`, `new-idea-march`, `final_final_2`. Projects I was excited about, spent weeks on, and never shipped.

I used to think this was a motivation problem. Or a skill gap. It wasn't either.

## The Real Problem

Every unshipped project died the same way: I kept building instead of releasing.

There was always one more feature to add. The UI wasn't quite right. The mobile experience needed work. The name wasn't finalized. I wanted to add analytics first.

**Perfectionism isn't a virtue. It's a trap.**

The project isn't real until someone else uses it. No matter how good it looks in localhost.

## What Changed

I set a rule for myself: **ship within 7 days of starting.**

That's it. 7 days. Whatever state it's in.

This forces brutal prioritization. You can't build everything in 7 days. So you ask: what is the one thing this needs to be useful? Build that. Ship.

## FocusLoop was a 5-Day Build

The core of FocusLoop — 30-day challenge, task completion, streaks — took 5 days to build and ship.

It's not perfect. The design went through 4 iterations after launch. Some features were added post-ship. That's fine.

What matters is that it existed in the world on day 5. Real people could use it. I could get feedback.

## The "But It's Not Ready" Lie

Nothing is ever ready. Gmail was in beta for *five years*. YouTube launched with 18 videos.

"Not ready" is usually code for "I'm scared of what people will think."

Ship it anyway. The fear doesn't go away. But you get better at shipping in spite of it.

## v1 Rules

- One core feature only
- Ugly is fine
- Bugs that don't break the core flow: ship with them, fix later
- No analytics until you have users
- No custom domain until you have traction

Ship fast. Iterate in public. That's the whole game.
